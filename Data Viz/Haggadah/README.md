# The Haggadah, Measured

A text-analytic portrait of the Passover seder. Every word count, citation count, psalm cross-reference, and voice attribution is computed from [Sefaria's](https://www.sefaria.org/Pesach_Haggadah) actual Pesach Haggadah text.

![Radial visualization of the Haggadah's 5,538 Hebrew words arranged by seder section, colored by speaker voice]

---

## What this is

The Passover Haggadah contains 5,538 Hebrew words spread across 37 subsections. This project pulls the full text from Sefaria's open-source API, runs computational analysis on it, and renders the results as an interactive radial visualization.

Five dimensions are encoded:

1. **Who is speaking** — Named rabbis (Eliezer, Akiva, Gamliel…), the collective "we," Psalmic/liturgical voice, children, God/Torah
2. **Who we're talking to** — First person (we/our/us), second person (you/Your — addressing God), third person (he/his — narrating)
3. **Scriptural citations** — Every שנאמר ("as it is stated") counted in the Hebrew
4. **Prophetic perfect** — Past tense used for present experience ("who *has* redeemed us")
5. **Composition era** — When each layer was written, from Psalms (~1000 BCE) to medieval piyyutim (~1400 CE)

## Files

### `haggadah_generate.py`
The main script. Run this to fetch fresh data and generate everything.

```bash
pip install requests
python haggadah_generate.py
```

**What it does:**

1. Fetches all 37 Haggadah sections from `sefaria.org/api/texts/`
2. Fetches Psalms 113–118 and 136 for cross-referencing
3. Fetches Deuteronomy 26 for the First Fruits cross-reference
4. Runs seven analysis passes on each section:
   - **Rabbi name detection** — scans English segments for sage names + discourse markers
   - **Psalm cross-reference** — n-gram overlap matching (3–8 word sequences) against source Psalms with nikkud stripped
   - **Aramaic detection** — counts Aramaic-specific markers (דא, לחמא, גדיא, השתא…)
   - **שנאמר counting** — regex on Hebrew text
   - **Pronoun analysis** — we/our/us, you/your/thou, he/his/him in English
   - **Prophetic perfect** — "has brought/redeemed/taken/done/given/chosen…"
   - **Dayenu structure** — אלו and דינו occurrence counts
5. Checks Deuteronomy 26:5-8 verbatim overlap with First Fruits
6. Prints a formatted analysis table
7. Saves `haggadah_data.json`
8. Generates `haggadah_visualization.html` with all data embedded

**Rate limiting:** 0.25s delay between API calls. Total fetch time ~15 seconds.

### `haggadah_visualization.html`
Generated by the Python script. A self-contained interactive radial visualization — no server, no API calls, no dependencies. Just open in a browser.

- 3200×3200 canvas with Sunwashed Earth palette (olive grove, desert rose, warm mustard, sage, linen cream)
- Bar height = Hebrew word count
- Bar fill colors = voice composition
- Thick colored arcs mark seder phases (Maggid, Hallel, Nirtzah…)
- Staggered labels with leader lines to avoid overlap
- Hover any section for full tooltip: voices, composition era, citations, pronouns, prophetic perfect
- Click to pin tooltip
- Expandable raw data table at bottom

### `haggadah_readable.html`
Standalone backup with pre-verified data already embedded. Same visualization, no Python required. Data sourced from a Sefaria-Export GitHub clone verified in April 2026.

### `haggadah_data.json`
Raw analysis output. Array of 37 objects, one per section:

```json
{
  "k": "First Fruits",
  "s": 1,
  "d": "19 citations — 1 per 24 words. 53% of Deut 26:5-8.",
  "he": 452,
  "rP": 0,
  "pP": 1,
  "aP": 0,
  "cit": 19,
  "we": 16,
  "you": 19,
  "he2": 12,
  "ppf": 1,
  "dayC": 0,
  "ilC": 0,
  "mx": {"rabbis": 55, "god": 30, "collective": 15},
  "ag": {"biblical": 25, "mishnaic": 25, "talmudic": 30, "geonic": 10}
}
```

Fields:
- `k` — section name
- `s` — seder phase (0=before, 1=maggid, 2=meal, 3=barech, 4=hallel, 5=nirtzah)
- `d` — description
- `he` — Hebrew word count (computed)
- `rP` — rabbi-attributed % (computed from name detection)
- `pP` — psalm text % (computed from n-gram cross-reference)
- `aP` — Aramaic % (computed from marker detection)
- `cit` — שנאמר citation count (computed)
- `we`, `you`, `he2` — pronoun counts (computed from English)
- `ppf` — prophetic perfect instances (computed)
- `dayC`, `ilC` — Dayenu-specific: דינו and אלו counts (computed)
- `mx` — voice composition (editorial, informed by computed %)
- `ag` — composition era (editorial)

## What's computed vs. editorial

| Dimension | Method | Source |
|---|---|---|
| Hebrew word counts | Exact count from text | **Computed** |
| English word counts | Exact count from text | **Computed** |
| Rabbi attribution % | Name detection in English segments | **Computed** |
| Psalm text % | N-gram overlap vs. source Psalms | **Computed** |
| Aramaic % | Aramaic marker frequency | **Computed** |
| שנאמר count | Regex on Hebrew | **Computed** |
| Pronoun counts | Regex on English | **Computed** |
| Prophetic perfect | Pattern matching on English | **Computed** |
| Dayenu structure | Regex on Hebrew | **Computed** |
| Deut 26 overlap | N-gram matching | **Computed** |
| Voice composition (mx) | Informed by computed %, assigned by section | **Editorial** |
| Composition era (ag) | Based on scholarly consensus | **Editorial** |

### Known false positive corrections
- **Echad Mi Yodea**: rabbi detection returns 100% because "rabbi" appears as a counted item, not a speaker. Corrected to 0%.
- **First Fruits, Kadesh, Birkat, Hallel 2nd**: Aramaic detection flags Hebrew words containing Aramaic-like letter patterns. Corrected to 0%.
- **Birkat Hamazon**: rabbi detection at 46% is inflated by formulaic attribution phrases. Reduced to 5% in voice mix.

## Data source

All text from [Sefaria](https://www.sefaria.org), an open-source library of Jewish texts. The API is public and supports CORS.

- Haggadah: `sefaria.org/api/texts/Pesach Haggadah, [section]`
- Psalms: `sefaria.org/api/texts/Psalms.[number]`
- Deuteronomy: `sefaria.org/api/texts/Deuteronomy.26`

The Sefaria-Export GitHub repository (`github.com/Sefaria/Sefaria-Export`) was used for initial development and verification via sparse checkout of `json/Liturgy/Haggadah/Pesach Haggadah/`, `json/Tanakh/Writings/Psalms/`, and `json/Tanakh/Torah/Deuteronomy/`.

## Context

Built for Passover 2026 preparation. The analysis was developed iteratively — starting from word counts, then adding cross-referencing, voice attribution, tense analysis, and repetition tracking as each layer revealed new patterns in the text.

The central insight: the Haggadah includes 5,538 words contain layers spanning 2,500 years — from Psalms composed in the First Temple period to Chad Gadya written around 1400 CE. The visualization makes this visible at a glance.
